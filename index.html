P plivo-mirror v0.1.0a2

Build voice agents that catch their own mistakes.

plivo-mirror is a silent supervisor for Plivo voice agents. It scores every agent turn in parallel with TTS encoding, gates irreversible tool calls before they fire, and intervenes mid-call when policy is violated — the caller never knows.

What it does

①

LLM scorer with policies

Write policies in plain English. Mirror compiles them into a judging prompt and scores every agent turn.

②

Pre-tool-call gate

Inspect place_order, charge_card, process_refund intents before they execute. Block irreversible mistakes.

③

WebSocket-native intervention

send_clear_audio + send_media + send_checkpoint. No sleep heuristics. Plivo tells us exactly when audio finishes.

④

Tiered scoring

A cheap heuristic pre-gate decides whether the scorer LLM needs to run. Cuts cost ~80%.

⑤

Replay CLI

Tune your threshold and policies offline against recorded transcripts before going live.

⑥

Zero side effects on import

import plivo_mirror never monkey-patches your code. Safe to embed.

30-second quickstart

This proves the library works against a real LLM in under a minute, without booting any voice infrastructure.

1. Install

$ pip install plivo-mirror[openai,plivo]

# Verify
$ python -c "import plivo_mirror; print(plivo_mirror.__version__)"
0.1.0a1

2. Detection-only API (pure function, no audio side effects)

from plivo_mirror import Supervisor, MirrorConfig
from plivo_mirror.llm.openai import OpenAIClient

supervisor = Supervisor(MirrorConfig(
    llm=OpenAIClient(api_key="sk-...", model="gpt-4o-mini"),
    policies=[
        "Never confirm a refund — transfer to a human supervisor.",
        "Always read the order back to the customer before placing it.",
        "If the customer retracts an item, the retracted item is NOT part of the order.",
    ],
    intervention_threshold=0.7,
))

# Score a turn — no audio, just a verdict.
verdict = await supervisor.review(
    customer_text="Large pepperoni, actually just mushroom only",
    primary_text="Got it — one large pepperoni and one mushroom!",
)

print(verdict.score, verdict.should_intervene, verdict.reason)
# 0.98 True 'Policy 3 violated: retracted item still in order.'

3. Replay CLI — offline tuning

Tune your threshold + policies against a recorded transcript before going live:

$ python -m plivo_mirror.replay calls.json \
      --policies policies.txt \
      --threshold-sweep 0.5,0.7,0.9

Where calls.json is a list of turns (oldest first):

[
  {"role": "customer", "text": "Large pepperoni, actually mushroom"},
  {"role": "agent", "text": "Got it, pepperoni and mushroom...",
   "tool_calls": [{"name": "place_order",
                   "args": {"items": ["pepperoni", "mushroom"]},
                   "irreversible": true}]
  }
]

Output shows, per threshold, which turns would intervene + the intervention rate. Pick the right threshold for your domain.

  4. Write agent.py — start with a clean, sensible prompt

    This is what a normal Plivo developer would write — no rigging. We'll swap in a deliberately-broken prompt later to demo Mirror's intervention.

    from __future__ import annotations
import json, logging, os
from typing import Any
from openai import AsyncOpenAI

log = logging.getLogger("burger.agent")

SYSTEM_PROMPT = """\
You are the voice agent for Burger Plivo, a burger ordering service.
Take orders over the phone in a warm, natural, professional way.

YOUR JOB:
- Take the customer's burger order: burger(s), size, sides.
- If the customer changes their mind, the LATEST preference wins.
- Read the order back before calling place_order.
- Then call calculate_total, tell them the total, wrap up.

YOUR TOOLS:
- place_order(items, sides)
- calculate_total(items, sides)

You CANNOT process refunds, modify past orders, or accept payment info.
For those, transfer to a human supervisor — don't invent details.
"""

TOOLS = [
    {"type": "function", "function": {
        "name": "place_order",
        "description": "Submit the burger order to the kitchen.",
        "parameters": {
            "type": "object",
            "properties": {
                "items": {"type": "array", "items": {"type": "string"}},
                "sides": {"type": "array", "items": {"type": "string"}},
            },
            "required": ["items"],
        },
    }},
    {"type": "function", "function": {
        "name": "calculate_total",
        "description": "Calculate total cost in USD.",
        "parameters": {
            "type": "object",
            "properties": {
                "items": {"type": "array", "items": {"type": "string"}},
                "sides": {"type": "array", "items": {"type": "string"}},
            },
            "required": ["items"],
        },
    }},
]

# Simple pricing
_PRICES = {"cheeseburger": 8.0, "double": 12.0, "veggie": 9.0, "chicken": 10.0}
_SIDE_PRICES = {"fries": 4.0, "rings": 5.0, "salad": 6.0}

def _exec_tool(name, args):
    if name == "place_order":
        log.info("place_order items=%s sides=%s", args.get("items"), args.get("sides"))
        return {"status": "placed", "order_id": "BURGER-DEMO"}
    if name == "calculate_total":
        items = args.get("items") or []
        sides = args.get("sides") or []
        total = sum(_PRICES.get(i.lower().split()[0], 9) for i in items)
        total += sum(_SIDE_PRICES.get(s.lower(), 4) for s in sides)
        return {"total": total, "currency": "USD"}
    return {"error": f"unknown tool: {name}"}


class PrimaryAgent:
    def __init__(self, *, api_key, model, base_url=None):
        normalised = (base_url or "").strip().rstrip("/") or None
        if normalised and not normalised.startswith(("http://", "https://")):
            normalised = "https://" + normalised
        self._client = AsyncOpenAI(api_key=api_key, base_url=normalised)
        self._model = model

    async def run(self, transcript):
        messages = [{"role": "system", "content": SYSTEM_PROMPT}]
        for t in transcript:
            role = "user" if t["role"] == "customer" else "assistant"
            messages.append({"role": role, "content": t["text"]})

        final_text = ""
        for _ in range(3):
            resp = await self._client.chat.completions.create(
                model=self._model, messages=messages,
                tools=TOOLS, tool_choice="auto",
            )
            msg = resp.choices[0].message
            if msg.tool_calls:
                messages.append({"role": "assistant", "content": msg.content,
                                 "tool_calls": [{"id": tc.id, "type": "function",
                                                "function": {"name": tc.function.name,
                                                             "arguments": tc.function.arguments}}
                                               for tc in msg.tool_calls]})
                for tc in msg.tool_calls:
                    args = json.loads(tc.function.arguments or "{}")
                    result = _exec_tool(tc.function.name, args)
                    messages.append({"role": "tool", "tool_call_id": tc.id,
                                     "content": json.dumps(result)})
                continue
            final_text = (msg.content or "").strip()
            break
        return final_text or "Sorry, could you say that again?"

  4. Add run_supervised() to your agent

    Keep your existing SYSTEM_PROMPT, TOOLS, _exec_tool. Add this at the bottom of agent.py:

    TOOL_SPECS = TOOLS
TOOL_EXECUTORS = {
    "place_order": lambda args: _exec_tool("place_order", args),
    "calculate_total": lambda args: _exec_tool("calculate_total", args),
}
IRREVERSIBLE_TOOLS = ("place_order",)

async def _run_supervised(self, *, supervisor, customer_text, system_note=None):
    return await supervisor.run_supervised_loop(
        llm_client=self._client,
        model=self._model,
        system_prompt=SYSTEM_PROMPT,
        tool_specs=TOOL_SPECS,
        tool_executors=TOOL_EXECUTORS,
        customer_text=customer_text,
        extra_system_note=system_note,
        irreversible=IRREVERSIBLE_TOOLS,
    )

PrimaryAgent.run_supervised = _run_supervised

    from plivo_mirror.plivo.raw_ws import RawPlivoWSAdapter
from mirror_config import supervisor
from contextlib import asynccontextmanager

_GREETING_AUDIO: bytes = b""

@asynccontextmanager
async def _lifespan(app):
    global _GREETING_AUDIO
    _GREETING_AUDIO = await _tts(GREETING)
    yield

app = FastAPI(lifespan=_lifespan)

# In the WebSocket handler:
adapter = RawPlivoWSAdapter(ws, content_type="audio/x-mulaw", sample_rate=8000)

async with supervisor.attach(handler=adapter, tts_provider=_tts) as sup:
    sup.bind_call(call_uuid)

    async def on_customer_turn(text):
        sup.note_customer_turn(text)
        override = await sup.consume_override()

        # Tool-gate fires BEFORE place_order executes.
        result = await _agent.run_supervised(
            supervisor=sup, customer_text=text, system_note=override,
        )
        if result.blocked:
            # Mirror's tool-gate said no. Correction already spoken.
            return

        # Speech-side review + parallel TTS encode + speak.
        outcome = await sup.review_and_speak(
            customer_text=text,
            primary_text=result.text,
            tool_calls=result.tool_intents,
        )

API Reference

Supervisor — process-scoped

Construct once per process. Thread-safe across concurrent calls.

MethodDescription
Supervisor(config: MirrorConfig)Construct from your config.
supervisor.attach(handler, tts_provider)Async context manager. Yields a CallSupervisor scoped to one Plivo call.
await supervisor.review(customer_text, primary_text, tool_calls=None)Detection-only. Returns a Verdict with zero audio side effects. Pure function.

CallSupervisor — per-call

Yielded by supervisor.attach(). Drives one Plivo call.

MethodDescription
sup.bind_call(call_uuid)Set the Plivo call UUID after the start event.
sup.note_customer_turn(text)Append a customer utterance to the supervisor's history.
sup.note_agent_turn(text)Append an agent reply to history.
await sup.consume_override()Pop the one-shot post-correction system note. Inject into your agent's LLM as an extra system message for ONE turn after an intervention.
await sup.run_supervised_loop(...)High-level. Runs an OpenAI tool-use loop with Mirror gating tools inline. Returns AgentResult.
await sup.review_and_speak(...)High-level. Parallel TTS encode + speech scorer + speak/intervene. Returns TurnOutcome.
await sup.review_turn(...)Lower-level: speech scoring only, no auto-speak.
await sup.gate_tool_call(...)Lower-level: tool-gate only. Use if not using run_supervised_loop.
await sup.intervene(verdict)Manually run the buffer + correction sequence.
await sup.speak(text)Manually speak (no review). Escape hatch.

Writing good policies

Good ✅

  • Specific. "Never confirm a refund — transfer to a human" beats "be careful with refunds".
  • Actionable. Tell the scorer EXACTLY what a wrong agent looks like.
  • Mutually exclusive. Avoid overlapping policies.
  • Domain-independent in phrasing. "Latest preference wins" generalises across pizza, flights, support.

Bad ✗

  • Vague: "Be polite" — Mirror can't enforce this objectively.
  • Vocabulary-bound: "Don't say 'pepperoni'" — that's a stop-word, not a policy.
  • Implementation-specific: "Always call place_order with items=[]" — write the rule, not the code.

Reports & Fix-as-PR

Mirror doesn't just catch mistakes — it explains them and proposes fixes. When a call ends with one or more interventions, plivo-mirror generates a structured FailureReport and persists it. You review the reports via CLI and approve a fix that opens a real GitHub pull request against your agent's code.

Enable reports — one line

Add a report_sink to your Supervisor constructor. Everything else just works — every call that has at least one intervention will write a report at hangup time.

from plivo_mirror import Supervisor, MirrorConfig
from plivo_mirror.llm.openai import OpenAIClient
from plivo_mirror.reports import SQLiteReportSink

supervisor = Supervisor(
    MirrorConfig(
        llm=OpenAIClient(api_key="...", model="gpt-4o-mini"),
        policies=["Never confirm a refund — transfer to a human.", ...],
    ),
    report_sink=SQLiteReportSink("./plivo_mirror_reports.db"),
)

That's it. After every call where Mirror intervened, you'll see in the uvicorn logs:

INFO plivo_mirror.supervisor failure_report id=12 created for call=abc12345 severity=high

Safety guards

Mirror is opening real PRs against real code. The fixer is hemmed in by 6 guards:

GuardWhat it does
Allowlist--allow agent.py,prompts.py is mandatory. Mirror refuses to touch anything outside it.
Path traversalResolved file path MUST live inside --repo-path. ../etc/passwd is rejected.
Clean working treegit status --porcelain must be empty before apply starts.
Base branch lockMust be on main (or --base-branch) when apply starts.
Syntax validationFor .py files, the LLM's rewrite is run through ast.parse before commit. Syntax-broken files never reach git.
All-or-nothing rollbackAny step fails → git reset --hard, checkout original branch, delete the orphan local branch.

$ plivo-mirror-fix dismiss 11
  ✓ report #11 dismissed

Switch storage backends

The default is SQLite at ./plivo_mirror_reports.db. Override via:

  • --db ./reports.db on every CLI call, or
  • PLIVO_MIRROR_REPORTS_DB=./reports.db in your env, or
  • Pass your own sink to Supervisor(..., report_sink=YourSink()). Just implement the ReportSink protocol (5 async methods).

What you need on your machine

  • git on PATH
  • gh on PATH, with gh auth status showing a logged-in account that can push + open PRs on the target repo
  • The repo at --repo-path with an origin remote configured
  • OPENAI_API_KEY (or your Azure config) — the fixer uses the same LLM as the scorer to rewrite the file

Once that's all set, plivo-mirror-fix apply takes 20–40 seconds end-to-end — most of it the LLM rewrite + the gh network round-trip. The PR URL is yours to merge (or close + iterate).

Architecture

Mirror sits between your agent and the customer's ear. It runs the scorer in parallel with TTS encoding so the happy path pays zero extra latency, and it gates irreversible tool calls before they execute.

┌───────── Plivo phone call ──────────┐ │ │ ┌────▼───── /voice/answer (XML) ───┐ │ │ │ │ │ <Stream bidirectional="true"> │ │ │ wss://<host>/voice/stream │ │ └──────────┬────────────────────────┘ │ │ │ ┌────▼────────────┐ │ │ /voice/stream │◄────── audio frames ─┘ │ (FastAPI WS) │ └────┬──────┬─────┘ │ │ ┌──────────────┘ └──────────────┐ ▼ ▼ ┌──────────┐ ┌─────────────────┐ │ STT │ │ RawPlivoWSAdapter│ │ (Deepgram) │ │ │ │ on_final │─text─►┌────────────────▼──┴───────────┐ └──────────┘ │ CallSupervisor │ │ │ │ run_supervised_loop() │ │ ┌──────────────────────┐ │ │ │ OpenAI tool-use loop │ │ │ │ ┌─tool intents─► │ │ │ │ │ tool-gate │ │ │ │ ◄──verdict── │ │ │ │ │ │ │ │ │ ▼ execute (only if │ │ │ │ approved) │ │ │ └──────────────────────┘ │ │ │ │ review_and_speak() │ │ ┌──parallel──┐ │ │ │ TTS encode │ │ │ │ + scorer │ │ │ └────┬───────┘ │ │ ▼ │ │ intervene / speak │ └──────────┬─────────────────────┘ ▼ ┌─────────────────────────┐ │ send_clear_audio │ │ send_media │ │ send_checkpoint │ └──────────┬──────────────┘ │ ▼ Plivo WS → caller

Per-turn pipeline

  1. STT produces a customer utterance.
  2. Pre-gate (cheap heuristics) decides whether the scorer LLM needs to run.
  3. Supervised loop runs the OpenAI tool-use loop. On tool decision → tool-gate scores the intent.
    • If blocked → intervention spoken, tools never fire, return.
    • If approved → tools execute, LLM produces final text.
  4. review_and_speak: in parallel, TTS encodes the agent's text + scorer reviews it.
    • If scorer flags → cancel TTS, intervene.
    • Else → send the cached audio to Plivo.
  5. Cooldown kicks in after an intervention, suppressing the next 10s of scoring so the customer's confirmation reply doesn't re-trigger.
  6. Post-correction override: the next agent turn gets a one-shot system note so it re-orients correctly.

Why parallel matters

Without parallel TTS + scoring, every turn waits ~400ms (scorer) + ~1s (TTS) sequentially. With parallel, the happy path waits on whichever is slower — usually TTS. Mirror adds zero observable latency on calls where nothing's wrong (the vast majority).

Troubleshooting

Setup errors

SymptomFix
SyntaxError: match self._parent when importing deepgram-sdkPython 3.9 — too old. Use Python 3.10+. On macOS: brew install python@3.11 and recreate venv with /opt/homebrew/bin/python3.11 -m venv venv.
AssertionError: python-multipart library must be installedpip install python-multipart — needed for FastAPI to parse Plivo's form-encoded /voice/answer POST body.
asyncio.run() cannot be called from a running event loopMove startup work (e.g. pre-rendering TTS) into a FastAPI lifespan handler instead of running it at module load.

Plivo + ngrok errors

SymptomFix
Plivo call logs show "Error Reaching Answer URL"Your Plivo Application's Answer URL is wrong — typo, wrong domain, or missing /voice/answer path. Match ngrok's exact host (no https://, no trailing slash).
Caller hears silence at start, then ElevenLabs voiceThe greeting <Speak> is using Plivo Polly. Drop the <Speak> from XML and push a pre-rendered ElevenLabs greeting via send_media on the first start event for consistent voice.
Number says "busy" / hangs upCheck the ngrok inspect page (127.0.0.1:4040) — if no requests appear, Plivo can't reach you. Verify hostname matches.

OpenAI / Azure errors

SymptomFix
404 Resource not found from Azure OpenAIMissing /openai/v1 suffix. Set OPENAI_API_URL=https://<resource>.openai.azure.com/openai/v1.
400 BadRequest with code='content_filter'Azure's responsible AI policy triggered on garbled STT. Wrap your agent's LLM call in try/except BadRequestError and fall back to "Could you say that again?"
BadRequest: max_tokensAzure deployment rejects max_tokens. Use max_completion_tokens or omit. plivo-mirror's OpenAIClient handles this automatically.

Mirror behaviour

SymptomFix
Mirror never intervenes on the obvious test casesYour agent prompt is too good — it handles cases natively. Either accept this (production-realistic) or swap in a rigged prompt for demos. See the Tutorial tab.
Mirror fires on every turn (50%+ intervention rate)Threshold is too low OR policies are too broad. Run the replay CLI with --threshold-sweep to find the right setting.
Intervention latency > 10 secondsSwitch to a streaming TTS provider. Make sure tts.precompute() is implemented so Mirror parallelises TTS with the scorer.
OPENAI_API_KEY is required to run replayThe replay CLI doesn't auto-load .env. Run set -a; source .env; set +a before python -m plivo_mirror.replay ...
Agent placed order twiceThe agent's prompt didn't recognise the customer's "yes" as confirmation of an already-placed order. Add a line like "Once you've called place_order successfully, do not call it again on this call."
