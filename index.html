What plivo-mirror does

You already have a Plivo voice agent — STT → LLM → TTS — running over a <Stream> WebSocket. Sometimes the agent gets it wrong: captures a retracted item, agrees to a refund it shouldn't, calls place_order with the wrong arguments.

plivo-mirror wraps your agent's loop. Every turn:

  1. The customer speaks → your STT produces text.
  2. Your agent's LLM decides what to say + what tools to call.
  3. Mirror's tool-gate inspects the tool intent BEFORE it fires. If your policy says "never confirm refunds without a human", the gate blocks process_refund and Mirror speaks a correction instead.
  4. If the tools are approved, they execute, and the LLM produces a final response.
  5. Mirror's speech scorer reviews the response in parallel with TTS encoding. If it violates policy, Mirror cancels the queued audio (send_clear_audio) and speaks a corrective question.
  6. On the next turn, Mirror passes a one-shot system note to your LLM so it re-orients correctly.

The caller never knows Mirror exists. They just hear "Sorry, let me make sure I got that right..." and a correction question whenever the agent was about to slip up.

30-second quickstart (no Plivo call needed)

This proves the library works against a real LLM in 30 seconds, without booting any voice infrastructure.

from plivo_mirror import Supervisor, MirrorConfig
from plivo_mirror.llm.openai import OpenAIClient

supervisor = Supervisor(MirrorConfig(
    llm=OpenAIClient(api_key="sk-...", model="gpt-4o-mini"),
    policies=[
        "Never confirm a refund — transfer to a human supervisor.",
        "Always read the order back to the customer before placing it.",
        "If the customer retracts an item, the retracted item is NOT part of the order.",
    ],
    intervention_threshold=0.7,
))

# Detection-only — no audio side effects. Pure function.
verdict = await supervisor.review(
    customer_text="Large pepperoni, actually just mushroom only",
    primary_text="Got it — one large pepperoni and one mushroom!",
)

print(verdict.score, verdict.should_intervene, verdict.reason)
# 0.98 True 'Policy 3 violated: retracted item still in order.'

Or run the offline replay CLI against a recorded transcript:

$ python -m plivo_mirror.replay transcript.json --policies policies.txt --threshold-sweep 0.5,0.7,0.9

Integrate into your Plivo voice agent (full walkthrough)

This is the full recipe. By the end you'll have a Plivo voice agent with:

  • Bidirectional <Stream> over WebSocket
  • Deepgram STT → your LLM → ElevenLabs TTS
  • plivo-mirror reviewing every turn, gating irreversible tools, intervening on policy violations

Step 1 — install deps

$ pip install plivo-mirror[openai,plivo]
$ pip install fastapi uvicorn deepgram-sdk elevenlabs python-dotenv

Step 2 — write your policies

Put them in a policies.txt file, one per line, in plain English:

Never confirm a refund — transfer to a human supervisor.
Always read the order back before calling place_order.
If the customer changes their mind, the LATEST stated preference wins.
Treat third-party preferences ('my wife wants X') as context, not the caller's order.
Do not promise specific delivery times.

Step 3 — build the Supervisor

# mirror_config.py
import os
from plivo_mirror import Supervisor, MirrorConfig
from plivo_mirror.llm.openai import OpenAIClient

POLICIES = [
    "Never confirm a refund — transfer to a human supervisor.",
    "Always read the order back before calling place_order.",
    "If the customer changes their mind, the LATEST stated preference wins.",
    "Treat third-party preferences as context, not the caller's order.",
    "Do not promise specific delivery times.",
]

supervisor = Supervisor(MirrorConfig(
    llm=OpenAIClient(
        api_key=os.environ["OPENAI_API_KEY"],
        model=os.getenv("OPENAI_MODEL", "gpt-4o-mini"),
        base_url=os.getenv("OPENAI_API_URL"),    # Azure auto-detected if set
    ),
    policies=POLICIES,
    intervention_threshold=0.7,
    irreversible_tools=["place_order", "charge_card", "process_refund"],
))

Step 4 — define your agent's tools

Tools are plain functions taking a single args: dict. plivo-mirror's tool-gate runs before these execute — you don't have to add any gating logic yourself.

# agent.py
from openai import AsyncOpenAI
from plivo_mirror.agents.openai_loop import AgentResult

SYSTEM_PROMPT = "You are a voice agent for ..."

TOOL_SPECS = [
    {
        "type": "function",
        "function": {
            "name": "place_order",
            "description": "Submit the customer's order to the kitchen.",
            "parameters": {
                "type": "object",
                "properties": {
                    "items": {"type": "array", "items": {"type": "string"}},
                },
                "required": ["items"],
            },
        },
    },
]

def _place_order(args: dict) -> dict:
    items = args.get("items") or []
    # your real implementation here
    return {"status": "placed", "order_id": "ORD-..."}

TOOL_EXECUTORS = {"place_order": _place_order}

class PrimaryAgent:
    def __init__(self, *, api_key, model, base_url=None):
        self._client = AsyncOpenAI(api_key=api_key, base_url=base_url)
        self._model = model

    async def run_supervised(self, *, supervisor, customer_text, system_note=None) -> AgentResult:
        return await supervisor.run_supervised_loop(
            llm_client=self._client,
            model=self._model,
            system_prompt=SYSTEM_PROMPT,
            tool_specs=TOOL_SPECS,
            tool_executors=TOOL_EXECUTORS,
            customer_text=customer_text,
            extra_system_note=system_note,
            irreversible=("place_order",),
        )

Step 5 — write the WebSocket handler

plivo-mirror exposes two high-level methods that do the heavy lifting per turn:

MethodWhat it does
sup.run_supervised_loop(...)Runs your agent's tool-use loop with Mirror's tool-gate inline. Returns an AgentResult. If result.blocked=True, Mirror already intervened.
sup.review_and_speak(...)Mirror's speech scorer + TTS encoding in parallel. On approval, sends the audio. On violation, cancels the TTS and speaks a correction.
# main.py — only the per-turn logic is shown; full file in the example.
from mirror_config import supervisor
from agent import PrimaryAgent

_agent = PrimaryAgent(...)
_tts = ElevenLabsTTS(...)

async def handle_call(ws):
    async with supervisor.attach(handler=adapter, tts_provider=_tts) as sup:
        sup.bind_call(call_uuid)

        async def on_customer_turn(text):
            sup.note_customer_turn(text)
            override = await sup.consume_override()

            # Agent + tool-gate run together. Irreversible tools fire ONLY after Mirror approves.
            result = await _agent.run_supervised(
                supervisor=sup,
                customer_text=text,
                system_note=override,
            )
            if result.blocked:
                # Mirror's tool-gate said no. Correction was already spoken.
                return

            # Speech scorer + TTS run in parallel; on approval the cached
            # audio plays without extra latency.
            outcome = await sup.review_and_speak(
                customer_text=text,
                primary_text=result.text,
                tool_calls=result.tool_intents,
            )

The full runnable example is in the repo at v2/voice_agents/pizza_plivo/.

Step 6 — wire Plivo

Plivo XML for the call's answer URL:

<Response>
  <Stream bidirectional="true" streamTimeout="86400" keepCallAlive="true"
          contentType="audio/x-mulaw;rate=8000">
    wss://your-host/voice/stream
  </Stream>
</Response>
Bidirectional is required. The Mirror intervention path needs send_clear_audio + send_media + send_checkpoint over the same WebSocket. Unidirectional streams fall back to PlivoRESTTTSSink which works but loses the clear_audio primitive.

Public API reference

Supervisor (process-scoped)

MethodDescription
Supervisor(config: MirrorConfig)Construct once per process.
supervisor.attach(handler, tts_provider)Async context manager. Yields a CallSupervisor scoped to one Plivo call.
await supervisor.review(...)Detection-only. Returns a Verdict with no audio side effects.

CallSupervisor (per-call)

MethodDescription
sup.bind_call(call_uuid)Set the Plivo call UUID after the start event.
sup.note_customer_turn(text)Append a customer utterance to the supervisor's history.
sup.note_agent_turn(text)Append an agent reply to history.
await sup.consume_override()Pop the one-shot post-correction system note. Pass to your agent's LLM as an extra system message for ONE turn after an intervention.
await sup.run_supervised_loop(...)Run an OpenAI tool-use loop with Mirror gating tools inline. Returns AgentResult.
await sup.review_and_speak(...)Parallel TTS + speech scorer + speak/intervene. Returns TurnOutcome.
await sup.review_turn(...)Lower-level: speech scoring only, no auto-speak. Use if you want to drive intervention yourself.
await sup.gate_tool_call(...)Lower-level: tool-gate only. Use if not using run_supervised_loop.
await sup.intervene(verdict)Manually run the buffer + correction sequence.
await sup.speak(text)Manually speak a string (no review). Last-resort escape hatch.

Writing good policies

Policies are plain English sentences. Mirror's scorer compiles them into a numbered list inside a judging prompt and asks the LLM to flag violations.

Good policies are…

  • Specific. "Never confirm a refund — transfer to a human" beats "be careful with refunds".
  • Actionable. The policy should tell the scorer EXACTLY what an agent doing the wrong thing looks like.
  • Mutually exclusive. Avoid policies that overlap — they make the scorer's verdict ambiguous.
  • Independent of vocabulary. "Latest preference wins on changes of mind" generalises across pizza, flights, support, anything.

Bad policies are…

  • Vague. "Be polite" — Mirror can't enforce this objectively.
  • Vocabulary-bound. "Don't say 'pepperoni'" — that's a stop-word, not a policy.
  • Implementation-specific. "Always call place_order with items=[]" — write the rule, not the code.

Tune your threshold

Record a few real calls, dump them as transcript JSON, and run the replay CLI with --threshold-sweep before going live. See below.

Replay CLI

$ python -m plivo_mirror.replay transcript.json \
    --policies policies.txt \
    --threshold-sweep 0.5,0.6,0.7,0.8,0.9

Transcript format — JSON list of turns, oldest first:

[
  {"role": "customer", "text": "Large pepperoni, actually mushroom"},
  {"role": "agent", "text": "Got it, pepperoni and mushroom...",
   "tool_calls": [{"name": "place_order", "args": {"items": ["pepperoni", "mushroom"]}}]
  }
]

Output shows, per threshold, which turns would intervene + the intervention rate. Pick the threshold with the right balance for your domain.

Troubleshooting

SymptomCause / fix
"Error Reaching Answer URL" in Plivo logsWrong PUBLIC_HOST in your Plivo Application config. Match the ngrok hostname exactly (no https://, no trailing slash).
404 Resource not found from OpenAIAzure base_url is missing the /openai/v1 suffix. Set OPENAI_API_URL=https://<resource>.openai.azure.com/openai/v1.
Greeting plays in a different voice than the restYou're using Plivo's <Speak> in the XML. Drop it and push a pre-rendered ElevenLabs greeting via send_media on the first start event.
Mirror never intervenes on the obvious test casesYour agent's prompt is already too good — it handles the cases natively. Mirror catches the rare slips, not the obvious ones. Use a slightly rigged demo prompt to exercise the intervention path.
Intervention latency > 10sSwitch to a streaming TTS provider, and make sure tts.precompute() is implemented so Mirror parallelises TTS with the scorer.
Azure content filter 400 errorsGarbled phone-audio STT sometimes trips Azure's responsible AI policy. Wrap your agent's LLM call in try/except BadRequestError(code='content_filter') with a "could you say that again" fallback.
