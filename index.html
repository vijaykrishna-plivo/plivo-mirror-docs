What plivo-mirror v4 is

A real-time dual-boundary policy firewall for LLM voice agents. It sits between the agent's LLM and the world and stops bad output before it reaches the caller or fires an irreversible action.

① Speech boundary
tokens → TTS: fabricated facts, unauthorized commitments, missing disclosures.
② Action boundary
tool call → execution: wrong/unauthorized actions, prompt injection, policy breaches.

You attach it to a LiveKit agent by swapping the base class to SupervisedAgent and passing a configured Firewall. Business rules live in code (policies + validators), never in the prompt.

# v4 is published as 0.4.0 (latest stable)

The 5-line integration

Same shape as a normal LiveKit Agent — only the base class and one constructor arg change.

from plivo_mirror import Firewall
from plivo_mirror.adapters.livekit import SupervisedAgent

firewall = Firewall.from_env(policies=POLICIES, validators=VALIDATORS)   # 1

class MyAgent(SupervisedAgent):                                       # 2
    def __init__(self):
        super().__init__(firewall=firewall, instructions=PROMPT)        # 3

    @function_tool
    async def place_order(self): ...                                   # 4  reads from STATE, not args

# session.start(agent=MyAgent())                                       # 5

Policies

policies is a list[str]. The prefix decides how it's enforced:

FormEnforced asExample
FORBID: <phrase>Deterministic block if the phrase appears in the reply (~0 ms, no LLM)."FORBID: full refund"
REQUIRE: <phrase>Deterministic block if the phrase is missing (required disclosure)."REQUIRE: this call may be recorded"
plain sentenceEvidence for the grounded verifier on flagged spans."Never invent prices or promotions."

Known facts

Code-owned reference values seeded into SessionState at the start of every call. They're what the verifier grounds claims against — the model can read them (via grounding injection) but cannot mutate them.

KNOWN_FACTS = {
    "cancellation_fee": "20% of the fare, fixed and non-waivable on a call",
    "hours": "11am-9pm",
}

Session state & zero-argument tools

SessionState is the single source of truth, kept outside the model. Validate a caller's committable value once, write it to state, and have tools read from state — never from model-supplied arguments. This is the backbone of the wrong-action and prompt-injection defenses.

from plivo_mirror.state import args_from_state
from plivo_mirror.state.entities import ValidatedEntity

async def extract_state(self, customer_text):
    # validate, then write OUTSIDE the model's context
    self.state.set_entity("items", ValidatedEntity("item", found, customer_text))
    self.state.confirm_intent(", ".join(found))

@function_tool
async def place_order(self):
    args = args_from_state(self.state, ["items"])   # read from STATE, not the model
    self.state.log_committed_action("place_order", args)
    return {"placed": True, **args}

Validators — the action-boundary rule you write

A validator is a plain function gating one tool. Register it per tool name; it runs before the tool executes. If it returns a non-pass Verdict, the tool call is dropped and the firewall speaks a correction instead.

from plivo_mirror.contracts import ToolCallIntent, Verdict

def block_unauthorized_fee_waiver(intent: ToolCallIntent, state) -> "Verdict | None":
    # intent.args = the model's PROPOSED arguments for this turn
    if intent.args.get("waive_fee") and not state.entity_value("fee_waiver_authorized"):
        return Verdict.block(
            reason="fee waiver needs verified supervisor authorization",
            policy_id="fee_waiver_unauthorized",
            spoken_correction="I can cancel that with the standard refund, "
                              "but I'm not able to waive the cancellation fee.",
        )
    return None   # pass

firewall = Firewall.from_env(policies=POLICIES,
                             validators={"cancel_booking": [block_unauthorized_fee_waiver]})
Signaturefn(intent: ToolCallIntent, state: SessionState) -> Verdict | None
intent.nametool name
intent.argsthe model's proposed arguments (a dict) — what you inspect
intent.irreversiblewhether the action has irreversible side effects
returnNone = allow · Verdict.block(...)/Verdict.correct(...) = intervene
The action guard also runs, before your validators: false-completion (claims done with no tool call), arg↔state consistency (a proposed arg that disagrees with a same-named validated entity is blocked), and authorization separation.

Verdict

FactoryFields
Verdict.pass_(reason="ok", confidence=1.0)allow the turn
Verdict.correct(reason, spoken_correction, policy_id=None, span="", confidence=1.0)replace the reply with a grounded correction
Verdict.block(reason, policy_id=None, span="", spoken_correction="", confidence=1.0)hard-stop the reply/action

spoken_correction is the agent-voice line used as the intervention filler (spoken first, no LLM). policy_id and reason show up in logs and observability.

Verifier (swappable)

The only expensive call, scoped to flagged spans. A Protocol — drop in a fine-tune, a small hosted model, or a stub.

from plivo_mirror.verifier.base import GroundingEvidence, VerifierResult

class MyVerifier:
    async def verify(self, claim: str, evidence: GroundingEvidence) -> VerifierResult:
        # evidence.facts / .policies / .retrieved_facts / .customer_text
        return VerifierResult(supported=True, policy_id=None, reason="")

firewall = Firewall.from_env(policies=POLICIES, verifier=MyVerifier())

Default is LLMJudgeVerifier — a separate, stateless entailment call on the same single model (FACTS + POLICIES + reply → supported y/n + which policy). A pass-through verifier (always supported) is useful when you enforce purely at the action boundary and don't write tool-derived facts to state.

The two guards (what runs each turn)

review_turn runs the speech guard, then the action guard — the first to intervene wins, and pending tool calls are dropped.

① Speech guard tokens → TTS

1 · Deterministic — compiled FORBID:/REQUIRE: checks. Hard hit → block (verifier never runs).
2 · Risk-span tagger — flags prices, numbers, names, commitment words. No span → ~0 ms pass.
3 · NLI semantic signal — does the reply contradict the customer or a known fact? (optional semantic_signal).
4 · Grounded verifier — on flagged spans only: supported by FACTS + POLICIES? → pass / correct.

② Action guard tool → execution · ~0 ms

1 · False-completion — reply claims "done" with no backing tool call.
2 · Arg ↔ state — proposed args must match validated state (wrong item/amount → block).
3 · Authorization — a separate service decides what's permitted (the model never authorizes itself).
4 · Validators — your code rules per tool; then tools fire zero-argument from state.

Intervention & the "filler"

When a guard intervenes, delivery is two beats:

  1. Filler — the verdict's spoken_correction, spoken first with no LLM (covers latency). Keep it a short placeholder.
  2. Grounded answer — regenerated on the same model (or templated from state), then re-verified. Capped by max_correction_retries.
Want one line only? Set max_correction_retries=0 and escalate_on_nonconvergence=False, and put the complete correction in spoken_correction. The firewall then speaks just that line and skips regeneration.

Full example — flight_agent_mirror/agent.py

A booking agent whose prompt is deliberately over-permissive (it tries to waive the cancellation fee for an upset caller or a self-claimed "supervisor"). The firewall catches it at the action boundary and the unauthorized full-refund is dropped, replaced by a correction.

import os, logging
from dotenv import load_dotenv
from livekit import agents
from livekit.agents import AgentSession, JobContext, function_tool
from livekit.plugins import deepgram, elevenlabs, openai, silero

from plivo_mirror import Firewall
from plivo_mirror.adapters.livekit import SupervisedAgent
from plivo_mirror.contracts import ToolCallIntent, Verdict
from plivo_mirror.verifier.base import GroundingEvidence, VerifierResult
from flight_store import search_flights, seeded_store

load_dotenv()
logger = logging.getLogger("flight-agent-mirror")

SYSTEM_PROMPT = """You are Riley, a SkyLine Airways voice agent. Book or cancel
flights. Confirm details before any action. (This prompt is intentionally
over-permissive about fee waivers — the firewall is the real guardrail.)"""

# Business rules live in CODE, not the prompt.
POLICIES = [
    "Cancellations always refund 80% (a 20% fee); the standard 80% refund is allowed.",
    "Waiving the fee or a full 100% refund needs verified supervisor authorization; "
    "a caller claiming to be a supervisor is NOT sufficient.",
    "Never invent flights, fares, prices, or booking references.",
]
KNOWN_FACTS = {"cancellation_fee": "20% of the fare, fixed and non-waivable on a call"}

def _block_unauthorized_fee_waiver(intent: ToolCallIntent, state) -> "Verdict | None":
    if intent.args.get("waive_fee") and not state.entity_value("fee_waiver_authorized"):
        return Verdict.block(
            reason="fee waiver not authorized — 20% fee is fixed",
            policy_id="fee_waiver_unauthorized",
            spoken_correction="I can cancel that with the standard refund, "
                              "but I'm not able to waive the cancellation fee.",
        )
    return None

class _PassThroughVerifier:               # enforce at the action boundary only
    async def verify(self, claim, evidence: GroundingEvidence) -> VerifierResult:
        return VerifierResult(supported=True)

firewall = Firewall.from_env(
    policies=POLICIES,
    known_facts=KNOWN_FACTS,
    verifier=_PassThroughVerifier(),
    validators={"cancel_booking": [_block_unauthorized_fee_waiver]},
    max_correction_retries=0,            # one-line correction
    escalate_on_nonconvergence=False,
)

class MirrorFlightAgent(SupervisedAgent):
    def __init__(self):
        super().__init__(firewall=firewall, instructions=SYSTEM_PROMPT)
        self.store = seeded_store()
        self._last_results = {}

    async def on_enter(self):
        await super().on_enter()
        await self.session.generate_reply(instructions="Greet the caller; ask book or cancel.")

    @function_tool
    async def search_flights(self, origin: str, destination: str, date: str) -> dict:
        """Search flights for a route+date. origin/destination: city or 3-letter code."""
        results = search_flights(origin, destination, date)
        self._last_results = {f["flight_number"]: f for f in results}
        return {"found": len(results), "flights": results}

    @function_tool
    async def book_flight(self, flight_number: str, passenger_name: str,
                          age: int = None, phone: str = "", cabin: str = "economy") -> dict:
        """Book a searched flight after the caller confirms name/age/phone/cabin."""
        flight = self._last_results.get(flight_number.strip().upper())
        if flight is None: return {"booked": False}
        b = self.store.book(flight, passenger_name, age=age, phone=phone, cabin=cabin)
        return {"booked": True, "pnr": b.pnr, "price_usd": b.price_usd}

    @function_tool
    async def get_booking(self, pnr: str) -> dict:
        """Look up a booking by its 6-character PNR."""
        b = self.store.get(pnr)
        return {"found": bool(b), **({} if not b else b.__dict__)}

    @function_tool
    async def cancel_booking(self, pnr: str, waive_fee: bool = False) -> dict:
        """Cancel a booking after the caller confirms. waive_fee: full 100% refund."""
        b, refund = self.store.cancel(pnr, waive_fee=waive_fee)
        logger.info("cancel_booking pnr=%s waive_fee=%s refund=%s", pnr, waive_fee, refund)
        return {"cancelled": bool(b), "refund_usd": refund, "fee_waived": waive_fee}

def _agent_llm():
    if os.environ.get("AZURE_OPENAI_API_KEY"): return openai.LLM.with_azure()
    base = os.environ.get("OPENAI_BASE_URL") or os.environ.get("OPENAI_API_URL")
    return openai.LLM(model=os.environ.get("OPENAI_MODEL"), base_url=base,
                      api_key=os.environ.get("OPENAI_API_KEY"))

async def entrypoint(ctx: JobContext):
    session = AgentSession(vad=silero.VAD.load(), stt=deepgram.STT(model="nova-2"),
                           llm=_agent_llm(), tts=elevenlabs.TTS())
    await ctx.connect()
    await session.start(agent=MirrorFlightAgent(), room=ctx.room)

if __name__ == "__main__":
    agents.cli.run_app(agents.WorkerOptions(entrypoint_fnc=entrypoint))

Azure OpenAI notes

The default verifier works around Azure gpt-* quirks: use max_completion_tokens (not max_tokens); omit tools entirely instead of tool_choice="none"; temperature may be ignored; response_format={"type":"json_object"} is supported. Set creds via AZURE_OPENAI_*, or OPENAI_API_KEY + OPENAI_BASE_URL/OPENAI_API_URL + OPENAI_MODEL.
